# DevOps — Telegram AI-агент для администрирования серверов

Telegram-бот, который выполняет DevOps-задачи из чата с помощью LLM. Бот понимает запросы на естественном языке, сам подбирает и вызывает инструменты (выполнение команд, чтение/правка файлов, веб-поиск, работа с проектом и Kubernetes) и возвращает краткий итог.

Работает с любым **OpenAI-совместимым** API (RouterAI, OpenAI, Codex Sale и др.) и поддерживает несколько провайдеров и моделей одновременно.

---

## Возможности

**Выполнение задач**
- Команды на главном сервере (внутри контейнера бота) и на удалённых SSH-серверах.
- Чтение и запись файлов в рабочей директории (с автоматическим `.bak` и защитой от выхода за пределы каталога).
- Веб-поиск и загрузка веб-страниц для диагностики.
- Голосовые сообщения (транскрибация) и изображения/скриншоты (анализ).

**ИИ-агенты и провайдеры**
- Несколько провайдеров API и несколько агентов (моделей) с переключением активного.
- Режим совместной работы: агенты-советники дают рекомендации перед выполнением, ведущий агент принимает итоговое решение.

**Проекты**
- Привязка «сервер + рабочая директория + инструкции» к проекту.
- Инструменты по проекту: список файлов, поиск, чтение, `git apply` патчей, `git diff`, запуск тестов.
- Настройки деплоя (Docker build/push, `kubectl set image` + `rollout status`).

**Мониторинг**
- Кнопки CPU, RAM, диск, процессы, сеть, аптайм, система; графики нагрузки по всем серверам.
- Фоновый монитор: алерты администраторам о падении Docker-контейнеров, упавших systemd-юнитах и проблемах в Kubernetes.

**Kubernetes**
- Статус deployment, поды, логи, события; смена image, ConfigMap, Secret, перезапуск deployment.
- ИИ-планирование изменений K8s с подтверждением перед применением.

**Безопасность**
- Авторизация по Telegram ID: администраторы (`ADMIN_IDS`), разрешённые пользователи или открытый доступ (`PUBLIC_ACCESS`).
- Пароли SSH-серверов шифруются на диске (Fernet).
- TOFU-верификация SSH-хостов (`known_hosts`) с защитой от подмены ключа (возможный MITM).
- Чёрный список опасных команд + запрос подтверждения для рискованных действий и записи файлов.
- Кнопка ⏹ СТОП для аварийной остановки текущей задачи.
- Изоляция данных: не-администраторы получают отдельные конфиги (`data/users/<id>/`).
- SOCKS5-прокси для Telegram, API и веб-поиска.

## Требования

- Linux-сервер с Docker и плагином Docker Compose.
- Telegram Bot Token от [@BotFather](https://t.me/BotFather).
- API-ключ любого OpenAI-совместимого провайдера (по умолчанию [RouterAI](https://routerai.ru)).

## Установка

```bash
cd DevOps
./install.sh        # создаст .env и data/ из примеров
```

Заполните `.env`:

```env
# Обязательное
TELEGRAM_TOKEN=ваш_telegram_bot_token
AI_API_KEY=ваш_api_ключ
ADMIN_IDS=ваш_telegram_id            # несколько — через запятую/пробел

# Провайдер и модель (значения по умолчанию)
AI_BASE_URL=https://routerai.ru/api/v1
AI_MODEL=anthropic/claude-opus-4-7

# Доступ
PUBLIC_ACCESS=0                       # 1 — открыть бота всем

# Шифрование секретов серверов (задайте свой постоянный ключ)
SERVER_PASSWORD_KEY=случайная_длинная_строка
```

Запуск:

```bash
docker compose up -d --build
```

Логи и остановка:

```bash
docker compose logs -f --tail=100
docker compose down
```

## Переменные окружения

| Переменная | По умолчанию | Назначение |
|---|---|---|
| `TELEGRAM_TOKEN` | — | **Обязательно.** Токен бота |
| `AI_API_KEY` | — | **Обязательно.** Ключ API (синонимы: `ROUTERAI_API_KEY`, `OPENAI_API_KEY`, `CODEX_API_KEY`) |
| `AI_BASE_URL` | `https://routerai.ru/api/v1` | Base URL провайдера (синонимы: `*_BASE_URL`) |
| `AI_MODEL` | `anthropic/claude-opus-4-7` | Модель по умолчанию (синоним: `MODEL`) |
| `ADMIN_IDS` | — | Telegram ID администраторов |
| `PUBLIC_ACCESS` | `0` | Открытый доступ к боту |
| `SERVER_PASSWORD_KEY` | — | Ключ шифрования паролей серверов (синоним: `FERNET_KEY`; иначе берётся токен) |
| `DATA_DIR` | `/app/data` | Каталог данных |
| `WORK_DIR` | `/app` | Рабочая директория для файловых инструментов |
| `MAX_OUTPUT` | `4000` | Лимит вывода команды (символы) |
| `MAX_FILE_SIZE` | `200000` | Лимит размера записываемого файла (байты) |
| `MAX_HISTORY_CHARS` | `60000` | Лимит истории диалога |
| `AUTO_CONTINUE_LIMIT` | `3` | Автопродолжений при «промежуточных» ответах |
| `MAX_AGENT_STEPS` | `20` | Лимит шагов агента за задачу |
| `SESSION_TTL_DAYS` | `14` | Срок хранения сессий |
| `SERVICE_MONITOR_ENABLED` | `1` | Фоновый мониторинг падений |
| `SERVICE_MONITOR_INTERVAL` | `60` | Интервал мониторинга (сек) |
| `SERVICE_MONITOR_IGNORE` | — | Игнорируемые systemd-юниты в алертах (через запятую) |
| `CMD_TIMEOUT` | `180` | Таймаут выполнения команды (локально и по SSH), сек |
| `SSH_CONNECT_TIMEOUT` | `15` | Таймаут установки SSH-соединения, сек |
| `CLOUDFLARE_API_TOKEN` | — | Токен Cloudflare API (Zone:Read + DNS:Edit) |
| `REMNAWAVE_URL` | — | URL панели Remnawave |
| `REMNAWAVE_TOKEN` | — | API-токен Remnawave (раздел API Tokens) |
| `REMNAWAVE_COOKIE` | — | Кука защиты панели (реверс-прокси Caddy/eGames), формат `имя=значение` |
| `BACKUP_RETENTION` | `5` | Сколько бэкапов хранить |

Многие лимиты также меняются администратором прямо в боте (⚙️ Настройки → 🛠 Админ панель).

## Серверы

Серверы добавляются прямо в боте (🌐 Серверы → ➕ Добавить): имя, хост, пользователь, пароль или приватный SSH-ключ. Пароли и passphrase шифруются при сохранении в `data/servers.json`.

«Главный сервер» — это сам контейнер бота; команды на нём выполняются локально, без SSH, и доступны только администраторам.

## Безопасность данных

Не публикуйте и не передавайте файлы, в которых могут быть токены, ключи, пароли, прокси и SSH-доступы:

- `.env`
- `data/servers.json`, `data/proxies.json`, `data/sessions.json`
- `data/preapproved_commands.json`, `data/kubeconfig.yaml`
- `data/.ssh/`, `data/users/`
- `k8s/secret.yaml`

Эти пути уже включены в `.gitignore`. Для распространения используйте только примеры `.env.example` и `*.example.*`. Команда 📦 Бекап бота создаёт архив **без** `.env` и каталога `data/`.

## Cloudflare

Управление DNS и кэшем Cloudflare — кнопка ☁️ **Cloudflare** в меню и инструменты ИИ-агента.

Настройка: создайте API Token (Cloudflare → My Profile → API Tokens → шаблон «Edit zone DNS», права `Zone:Read` + `DNS:Edit`) и задайте его через `CLOUDFLARE_API_TOKEN` в `.env` либо в боте: ☁️ Cloudflare → 🔑 Задать токен.

Возможности:
- список зон (доменов) и DNS-записей;
- создание/изменение записей A, AAAA, CNAME, TXT (вкл. проксирование и TTL);
- удаление записей;
- полная очистка кэша зоны.

Через ИИ-агента можно командой на русском: «переключи `panel.darkprincepanel.ru` на IP нового сервера». Изменение/удаление записей и очистка кэша требуют подтверждения.

## Remnawave

Управление VPN-панелью [Remnawave](https://remna.st) — кнопка 🛰 **Remnawave** и инструменты ИИ-агента.

Настройка: создайте API-токен в панели (раздел API Tokens) и задайте `REMNAWAVE_URL` + `REMNAWAVE_TOKEN` в `.env`, либо в боте: 🛰 Remnawave → 🔑 Подключить панель (формат `URL | TOKEN | COOKIE`).

Если панель защищена кукой реверс-прокси (скрипты установки eGames/Caddy), укажите её третьим полем (`имя=значение`) или через `REMNAWAVE_COOKIE` — бот будет слать её в заголовке `Cookie` на каждый запрос.

Меню (чтение): статистика, список пользователей, нод и хостов.

ИИ-агент (инструменты `rw_stats`, `rw_list_*`, `rw_api`): полный доступ к REST API панели — создание/изменение пользователей, нод, хостов и правка Xray config-профилей через универсальный `rw_api`. Команды на русском, например «покажи кто онлайн», «создай пользователя на 30 дней», «перезапусти ноду X». Любые операции записи (POST/PATCH/PUT/DELETE) требуют подтверждения.

## Kubernetes

Манифесты — в каталоге `k8s/`.

1. Соберите и опубликуйте Docker-образ, укажите его в `k8s/deployment.yaml`.
2. Скопируйте `k8s/secret.example.yaml` → `k8s/secret.yaml`, впишите `TELEGRAM_TOKEN` и `AI_API_KEY`.
3. При необходимости поправьте `k8s/configmap.yaml` (`ADMIN_IDS`, модель и т.д.).
4. Примените:

```bash
kubectl apply -k k8s/
```

Данные хранятся в PVC `devops-bot-data`. Для Telegram long-polling Service/Ingress не нужен.

Управление кластером из бота (кнопка ☸️ Kubernetes или команды):

```text
/k8s_config KEY VALUE   # обновить ConfigMap и перезапустить deployment
/k8s_secret KEY VALUE   # обновить Secret и перезапустить deployment
/k8s_image IMAGE        # сменить Docker image deployment
```

Namespace/deployment/configmap/secret по умолчанию задаются переменными `K8S_NAMESPACE`, `K8S_DEPLOYMENT`, `K8S_CONFIGMAP`, `K8S_SECRET`.
